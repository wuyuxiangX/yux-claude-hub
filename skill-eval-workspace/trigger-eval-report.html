All 85 should-NOT-trigger test cases passed across every skill. The "Do NOT use for..." clauses, mutual cross-references, and pipeline hints are working exactly as designed. No false positives detected.

True Positive rate artificially low due to eval infrastructure

The 8.9% TP rate does not reflect real-world triggering. The eval framework creates a temporary command that competes with the already-installed plugin skill. Claude consistently chooses the installed skill over the temp command. In practice, users will have the skills installed, so the installed path IS the correct trigger path — the eval just can't measure it.

blog-oss had the best TP rate (25%)
